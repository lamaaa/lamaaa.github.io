数据库学习笔记二（数据库设计与E-R模型）

Posted on 2016-11-27 | In Technology , Database |
  1. 参与联系集的实体集的数目成为联系集的度（degree）。
  2. E-R模型中的属性可以按照如下的属性类型来进行划。
    • 简单和复合：简单指的是不能划分为更小的部分。复合指的是可以再划分的部分。
    • 单值和多值：
    • 派生：这类属性的值可以从别的相关属性或实体派生出来。派生属性的值不存储，而是在需要时计算出来
    • 当实体在某个属性上没有值时使用空值。空值可以表示“不适用”，即该实体的这个属性不存在值。或者用来表示属性值未知。未知的值可以是缺失的，或不知道的。
  3. 映射基数，或基数比率，表示一个实体通过一个联系集能关联到的实体的个数。
    • 一对一。A中的一个实体至多与B中的一个实体相关联，并且B中的一个实体也至多与A中的一个实体相关联
    • 一对多。A中的一个实体可以与B中的任意数目实体相关联，而B中的一个实体至多与A中的一个实体A相关联
    • 多对一。A中的一个实体至多可以与B中的一个实体相关联。而B中的一个实体可以与A中的任意数目实体相关联
    • 多对多。A中的一个实体可以与B中的任意数目的实体相关联。而B中的一个实体也可以与A中的任意数目的实体相关联
  4. 参与约束，如果实体集E的每个实体都参与到联系集R的至少一个联系中，实体集E在联系集R中的参与称为全部的，如果实体集E只有部分实体参与到R的联系中，实体集E到联系集R的参与称为是部分的。
  5. 弱实体集，没有足够的属性以形成主码的实体集称为弱实体集，有主码的实体集称为强实体集。

数据库学习笔记一（某些定义）

Posted on 2016-09-18 | In Technology , Database |

关系数据库的结构

1.关系用来指代表，而元组用来指代行，属性指代表中的列。

2.对于关系的每个属性，都存在一个允许取值的集合，称为该属性的域。域是原子的，不可再分。

数据库模式

3.数据库模式是数据库的设计逻辑，数据库实例是给定时刻数据库中数据的一个快照。

码

4.超码是一个或多个属性的集合，这些属性的组合可以使我们在一个关系中唯一地标识一个元祖。如果K是一个超码，那么K的任意超集也是超码。

5.如果一个超码它的任意真子集都不能成为超码。这样的最小超码称为候选码。

6.候选码可能有多个，我们用主码这个术语来代表被数据库设计者选中的、主要用来在一个关系中区分不同元组的候选码。码（不论是主码、候选码或超码）是整个关系中的一种性质，而不是单个元组的性质。

7.一个关系模式（如r1）可能在它的属性中包括了另一个关系模式（如r2）的主码。这个属性在r1上称作参照r2的外码。关系r1也称为外码依赖的参照关系。r2叫做外码的被参照关系。

8.参照完整性约束要求在参照关系中任意元组在特定属性上的取值必然等于被参照关系中某个元组在特定属性上的取值。

抽象类和接口

Posted on 2016-08-02 | In Technology , PHP |

以前对于抽象类的理解总是懵懵懂懂，半桶水。如今借着《深入PHP面向对象、模式与实践》，加上自己Google之后，有了一些新的理解。嗯好记性不如烂笔头，趁着没忘记之前，我要记下来。

抽象类

这里就用Java抽象类的定义吧，反正学习的是面向对象的思想，哪门语言并不是特别重要。Java抽象类：使用了关键词abstract声明的类叫作“抽象类”。如果一个类里包含了一个或多个抽象方法，类就必须指定成abstract（抽象）。“抽象方法”,属于一种不完整的方法，只含有一个声明，没有方法主体。

所以为什么要有抽象类呢？为什么不直接实现呢？
首先，抽象类在Java语言中表示的是一种继承关系，一个类只能使用一次继承关系。有时候我们需要定义一个基类，然后衍生出几个子类。不同子类是不一样的，可能对于同一个继承的基类方法，实现并不一样。并且我们不需要实例化这个基类，但是有一点可以确定的是，所有的子类都需要实现这个方法。既然如此，那基类中写什么代码并不重要，甚至不写也可以，也就是空方法。而这个基类就是抽象类。
这个有一个例子。感觉这个例子解释地很不错。

接口

接口：Java接口是一系列方法的声明，是一些方法特征的集合，一个接口只有方法的特征没有方法的实现，因此这些方法可以在不同的地方被不同的类实现，而这些实现可以具有不同的行为（功能）。

为什么要有接口呢？
第一，接口是一个规范。很多时候很多类会去实现同一个接口，大家需要统一标准，而统一标准的目的就是让大家知道这个是做什么的，但是不知道具体怎么做（各个类有各个类的实现方法）
着也有一个例子,解释地生动形象。

还有一点是接口在开发过程中可以快速分离工作内容。
比如调用者在写业务逻辑的时候需要一个功能，可能是数据库访问，或者复杂计算，但是他的工作专注于实现业务逻辑，不想分开精力去做底层实现，那么他只需要先实现一个接口，定义了规范，然后就可以继续他的业务逻辑代码了。而实现者可以根据这个接口规范，做具体的实现。这样通过使用接口就可以快速的分离工作内容，达到团队并行工作

接口和抽象类

抽象类和接口有类似之处，都不能用于创建对象，接口中的方法其实都是抽象方法。如果抽象类中只定义了抽象方法，那抽象类和接口就更像了。但抽象类和接口根本上是不同的，一个类可以实现多个接口，但只能继承一个类。

抽象类和接口是配合而非替代关系，它们经常一起使用，接口声明能力，抽象类提供默认实现，实现全部或部分方法，一个接口经常有一个对应的抽象类。

Ubuntu系统配置Apache虚拟主机

Posted on 2016-07-31 | In Technology , Tutorial |

假设在我的网站上www.linhaoyang.com建立了三个系统，分别是系统1，系统2和系统3。对应的二级域名分别是1.linhaoyang.com,2.linhaoyang.com和3.linhaoyang.com。
首先建立一个根目录文件wwwRoot,再分别建立三个目录对应这三个系统。
sudo mkdir -p wwwRoot/{1,2,3}
在我的服务器上，apache的配置文件在/etc/apache2/下，而虚拟主机的配置文件则是在apache2/sites-avaiable/中

其中，000-default.conf文件是apache的默认虚拟主机配置文件，于是我们只要将这个文件复制修改，就可以得到其中虚拟主机的配置文件。
sudo cp 000-default.conf 1.conf
sudo cp 000-default.conf 2.conf
sudo cp 000-default.conf 3.conf
然后分别修改文件,添加ServerName和修改DocumentRoot：
sudo vim 1.conf

其他两个文件2.conf和3.conf也是做类似的修改。
修改完之后，我们知道，这只是apache可用的虚拟主机，而我们还需要启用这三个虚拟主机。需要切换到sites-enabled文件中建立软链接。即：
sudo cd ../sites-enabled/
sudo ln -s ../sites-avaiable/1.conf 1.conf
sudo ln -s ../sites-avaiable/2.conf 2.conf
sudo ln -s ../sites-avaiable/3.conf 3.conf

这样我们就建立了三台虚拟主机，但是它并不会马上用于配置中，我们还需要启用配置，然后再重启apache。
启用配置：
sudo a2ensite /etc/apache2/sites-avaiable/1.conf
sudo a2ensite /etc/apache2/sites-avaiable/2.conf
sudo a2ensite /etc/apache2/sites-avaiable/3.conf
重启apache：
sudo service apache2 restart
此时若是访问这三个网站，便会发现403Forbidden的错误，很明显这是权限问题。在/etc/apache2/appache2.conf文件中添加这些配置即可解决。
sudo vim /etc/apache2/apache2.conf

同时要保证/var/wwwRoot/1/,/var/wwwRoot/2/和/var/wwwRoot/3/三个文件夹都具有755的权限即可。
最后再次重启apache便大功告成。

Ubuntu系统搭建LAMP环境

Posted on 2016-07-30 | In Technology , Tutorial |

LAMP是指一组通常一起使用来运行动态网站或者服务器的自由软件名称首字母缩写：

  • Linux系统，操作系统
  • Apache，Web服务器
  • Mysql，数据库管理系统
  • PHP，脚本语言


用户通过浏览器输入网址，DNS服务器将网址转换成IP地址找到服务器，来到apache这里，apache内部有很多可扩展的模块。其中，有一个叫做libphp5.so的模块，它可以指挥php引擎工作。apache将浏览器请求的代码交给php解析器进行解析。php解析器内部也有很多可扩展的模块，其中有一个叫做mysql.so的模块，它可以指挥mysql进行CURD操作（但并不是每次都要进行mysql的CURD操作）。然后PHP解析器将处理后的数据返回给apache，而apache将相应数据返回给浏览器，浏览器再将页面展示给用户。

使用apt-get工具安装Apache、PHP、MySQL

  1. 首先安装安装apache
    sudo apt-get install apache2
    查看apache的版本
    apache2 -v
  2. 接着是安装php
    sudo apt-get install php5
    查看php的版本
    php5 -v
  3. 检查apache是否加载libphp5.so
    cat /etc/apache2/mods-enabled/php5.load
    看到以下的内容，说明apache已经成功加载了libphp5.so这个模块
    LoadModule php5_module /usr/lib/apache2/modules/libphp5.so
    sudo apt-get install mysql-server
  4. 检查php是否加载mysql.so
    cat /etc/php5/mods-avaiable/mysql.ini
    你会看到No such file or directory，这是因为php在安装的过程中默认不安装mysql的扩展，所以我们需要手动地安装。
    sudo apt-get install php5-mysql
    cat /etc/php5/mods-avaiable/mysql.ini
    当你看到以下内容时，说明php已经成功加载了mysql.so这个模块
    ; configuration for php MySQL module
    ; priority=20
    extension=mysql.so

为了确保它们能够正常的工作，需要重启一下mysql和apache
sudo service mysql restart
sudo service apache2 restart

当你熟悉如何搭建LAMP环境时，你可以用一条命令搞定上述这些步骤，即:
sudo apt-get install apache2 php5 mysql-server php5-mysql

创建phpinfo服务器探针

几个主要的组件都已经安装完毕，接下来在服务器上创建一个mysql连接测试文件和以及phpinfo的探针，来看看他们是否正常工作。
进入根目录/var/www/html/创建测试文件
cd /var/www/html
sudo vim info.php

然后在浏览器中查看测试结果。

给PHP添加常用拓展

根据服务器实际需求，我们可以选择性地安装一些PHP拓展包，比如gd包，curl包等等。
sudo apt-get install php5-gd curl libcurl3 libcurl3-dev php5-curl
安装完后为确保拓展包能够被正确加载，需要重启apache。
sudo service apache2 restart
检验是否成功加载：
回到php的探针页面，查看是否已经加载模块。

安装PHPMyAdmin

sudo apt-get install phpmyadmin
sudo ln -s /usr/share/phpmyadmin/ /var/www/html/pma
