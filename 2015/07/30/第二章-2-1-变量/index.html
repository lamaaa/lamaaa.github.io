Posted on 2015-07-30 | In Technology , C++ |

##变量

变量是一个具名的，可供程序操作的存储空间。对于C++程序员来说，“变量”和“对象”一般可以互换使用。变量定义的形式是由一个类型说明符紧跟一个或多个变量名组成的列表，其中变量名以逗号分隔，最后以分号结束。

当对象在创建时获得了一个特定的值，我们说这个对象被初始化了。用于初始化的值可以是任何复杂的表达式。初始化和赋值并不一样，初始化是在创建对象时赋予其一个一个初始值，而赋值的含义是把对象的当前值擦除，而以一个新值来代替。

###1.初始化：
C++定义了初始化的好几种形式，例如，想要定义一个变量名为units_sold的int型变量并初始化为0，以下4条语句都可以做到。

  1. int units_sold = 0;
  2. int units_sold = {0};
  3. int units_sold{0};
  4. int units_sold(0);

使用花括号来初始变量叫做列表初始化。当用于内置类型的变量时，这种初始化形式有一个重要特点：如果我们使用列表初始化并且初始值存在丢失信息的风险，则编译器报错。

####默认初始化：

如果定义变量时没有指定初值，则变量被默认初始化，此时变量被赋予了“默认值”。“默认值”是什么由变量类型决定，同时定义变量的位置也会对此有所影响。
如果内置类型未被显式初始化，它的值由定义位置决定。

  • 定义于任何函数体之外的变量初始为0，一种特殊情况是，定义在函数体内的内置类型将不被初始化，一个未被初始化的内置类型是未定义的。
  • 每个类决定其初始化对象的方式，而且，是否可以允许不经过初始化就定义对象也由类自己决定，如果允许这种行为，它将决定对象的初始值是什么。绝大多数类都支持无须显示初始化而定义对象，这样的类提供了一个合适的初始值。一些类要求对每个对象对显示初始化，如果定义对象没有显示初始化，则发生错误。

###2.变量声明和定义的关系（重要）
为了允许把一个程序拆成多个逻辑部分编写，C++语言支持分离式编译机制，该机制允许将程序分成多个文件，每个文件允许被独立编译。

为了支持分离式编译，C++语言将声明与定义分开来。声明使得名字为程序所知，一个文件如果想使用别处定义的名字则必须包含对那个名字的声明。定义负责创建与名字关联的实体。

变量声明规定了变量的类型和名字，这一点上定义与之相同。但是除此之外，定义还将申请存储空间，也可能会为变量赋一个初始值。

如果想声明一个变量而不去定义它，则在变量名前添加extern关键字，而且不要显式地初始化变量。任何包含了显式初始化的声明就变成了定义。我们能给由extern关键字标记的变量赋一个初始值，但是这么做也抵消了extern的作用。extern语句一旦包含了初始值就不再是声明，而变成了定义。

  • 在函数体内部，如果试图初始化一个由extern标记的变量，将引发错误。
  • 变量只能被定义一次，确可以被声明多次。

声明和定义的区别看起来微不足道，但实际上确非常重要。如果在多个文件中使用同一个变量，就必须将声明与定义分离，此时，变量的定义必须出现且只能出现在一个文件中，而其他要用到该变量的文件必须对其声明，且不能重复定义。

C++是以一种静态类型语言，其含义是在编译阶段检查类型。其中，检查类型的过程称为类型检查。
