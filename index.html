书单 | 推荐给计算机系学生的好书

Posted on 2017-05-17 | In Essay |

转眼间，大学已经度过三年了，这三年看过不少关于技术的好书，自己总结了一下，希望可以给有需要的人参考一下。以下按照时间顺序和模块来列出这些比较推荐的书籍吧。

C\C++

C\C++几乎是每位计算机系的学生在进入这个领域之后遇到的第一块最重要的基础，学好它的重要性不容置疑。
1.《The C Programming Language》，Brian W. Kernighan & Dennis M. Ritchie
这本书是C语言的作者写的，B5的开本，短短不到两百页，却在短小的篇幅里，手把手教你写了stdio.h stdlib.h string.h当中大部分例程，实现了二分查找、快速排序、二叉树、哈希表这些重要的数据结构和算法。

  1. 《C Primer Plus》，Stephen Prata/云巅工作室
    这本书和上面推荐的一样同是C语言的入门书，不一样的书的篇幅比较长。事无巨细，娓娓道来，它详细地讲述了C语言的各种细节。但是需要注意的是，这本书杂糅了一些不属于C语言的知识，比如它介绍了ADT、队列和二叉树，以及一些编译器、操作系统相关的知识，而这些东西都会在以后的课程慢慢学到。
  2. 《C++ Primer》，[美] Stanley B·Lippman / [美] Josee Lajoie / [美] Barbara E·Moo
    注意哦，这本是不带plus的哦（常常有同学看成带plus的那一本）。这本书和《The C Programming Language》一样是地位不可撼动的权威教材，非常经典。我自己曾经看了其中了一部分，感觉真实受益匪浅。但是更加建议有一些C\C++基础的读者看这本书，因为他定位的是程序员的C++教材，而不是计算机系新生的编程的入门书。还有就是不要只是简单地阅读而已，这本书有一本专门配套的习题答案，如果真心希望能吃透，还是要配合这本习题答案将书本上的每道题都弄懂。

数据结构与算法

数据结构与算法是程序员的安生立命之本。

  1. 《数据结构与算法分析》，[美] Mark Allen Weiss
    我知道很多大学用的是严蔚敏那本，不过这本讲的内容更多。它不在简单的地方无谓的浪费笔墨，恰到好处的把初学者带入算法和数据结构的世界。它用C语言实现，基本上涉及了数据结构基础的“方方面面”，当然这书的难度还是有的，如果看不懂的话，那就多看几遍，同时参考其他的书籍或视频来看。
  2. 《算法》，塞奇威克 / [美] 韦恩
    这书和上面介绍的《数据结构与算法分析》讲的内容差不多，它用Java语言实现，图解非常多，方便读者理解。

  3. 《算法导论》，[美] Thomas H·Cormen / Charles E·Leiserson / Ronald L·Rivest / Clifford Stein
    这书的出名程度已经不用我赘诉了。而我本人也没看过这本书。（心有余力不足啊，总想看但就是有各种其他的学习计划挡在前面，以后肯定会补回来的）。这里附上几点其他人的学习心得（来自知乎），篇幅关系，我缩减了不少：

    1.动手做。有些真正性能和实现上面的优点，不写代码是感受不出来的。2.尽量看英文原版（不是为了装逼），因为中文翻译差，不过也可以中英结合看。3.这书真不是导论，他的深度和难度都是大部分人难以驾驭的，学习路线很陡峭。4.配合其他资料一起看，啃硬书很寂寞啊。5.关于理论证明的取舍CLRS作为一本严谨的科学研究入门教材，理论的证明充斥了这本书的很大一部分。每一章节的很多题目，都是证明某个引理之类的。这一部分的取舍我觉得因人而异吧，有的同学只是为了掌握算法本身而去学CLRS，那么证明部分我觉得只要能做到说服自己即可；有的同学将来要读博发paper的话，CLRS上的证明最好还是弄透彻比较好。虽然自己再造一个什么新的排序算法，新的二叉搜索树什么的可能性不是很高（大神除外了，Size Balanced Tree人家中学时候造出来），但是严谨的数学素养是非常重要的。

  4. 《算法设计》，Jon Kleinberg / Eva Tardos
    这书同样相当经典，对比《算法导论》，《算法设计》更侧重算法设计思路，不再赘述算法复杂度的分析。建议先看算法导论再看这个书，颇有推理之旅的感觉。

计算机组成原理

  1. 《深入理解计算机系统》，[美] Randal E·Bryant / David O’Hallaron
    本书从程序员的视角详细阐述计算机系统的本质概念，并展示这些概念如何实实在在地影响应用程序的正确性、性能和实用性。全书共12章，主要内容包括信息的表示和处理、程序的机器级表示、处理器体系结构、优化程序性能、存储器层次结构、链接、异常控制流、虚拟存储器、系统级I/O、网络编程、并发编程等。书中提供大量的例子和练习，并给出部分答案，有助于读者加深对正文所述概念和知识的理解。

编译器

说来惭愧，作为程序员三大浪漫之一，博主的学校计算机本科没有开设编译原理这门课（不要问我为什么），但是下面的三本书还是了解的。

  1. 《Compilers: Principles,Techniques,and Tools》，Alfred V.Aho,Ravi Sethi,Jeffrey D.Ullman
    “龙书”。龙书是Alfred V.Aho等人于1986年出版的，由于出版年代较早，其中包含部分过时的技术并且没有反映一些新的编译技术。新编的《编译原理》抛弃诸如算符优先分析等过时技术，增加面向对象编译、类型检查等新技术。
  2. 《Modern Compiler Implementation in C》，Andrew W.Appel,with Jens Palsberg
    “虎书”。虎书出版比较晚，与《编译原理》的知识点差不多，但增加了数据流分析、循环优化、内存管理等内容。与虎书比，《编译原理》更适合国内的编译原理课程教学。这是C版，还有Java版和ML版。
  3. 《Advanced Compiler Design and Implementation》，Steven Muchnick
    “鲸书”。鲸书侧重在对编译器后端优化的处理。在本科阶段的编译教学中旨在让学生对程序设计语言的编译全过程有系统的理解，因此会介绍编译器后端的处理技术，但不注重优化技术。鲸书更适合作为研究生的教材或参考书。

计算机网络

  1. 《计算机网络：自顶向下方法》，[美] James F·Kurose / [美] Keith W·Ross
    这本书讲得通俗易懂，条理清晰，每个原理都特意用一个简单的例子来解释，感觉真是计网中里面不可多得的一本好书啊。博主自己的教材是自底向上的，但看了这本书后，发现自顶向下的方法会更加有助于我们理解整个计算机网络。强推！
  2. 《图解HTTP》，[日] 上野宣
    这本书深入浅出的讲解了HTTP协议及其实现原理，主要内容包括HTTP协议、报文格式、报文头结构、HTTPS原理、HTTP新增强的功能、HTTP的攻击等。由于附上了非常多的图解，非常直观易懂，是一本Web开发不可多得的入门书。

数据库

  1. 《数据库系统概念》[美] Abraham Silberschatz / [美] Henry F·Korth / [印尼] S·Sudarshan
    这本书比较适合初学者，每个概念讲得都非常详细，比较让人能够理解。不过中文翻译书还是有那个缺点，即语言不够精炼，所以有能力的同学还是看英文书吧。

PHP

博主的Web后台开发是从PHP开始的，之所以选择PHP的原因是PHP对于初学者非常简单，适合快速上手。不过学习PHP的过程必然涉及到Apache、MySQL的学习，所以应该是整个LAMP或者LNMP体系的学习。在有适合的基础之后，可以上手一两个MVC框架，入门建议Codeignter，之后可以学习一下Laravel（这个框架学习难度相对TP会更高，但是它的设计理念是非常先进的，值得学习的）。这些都比较了解后，个人建议可以自己造一个属于的MVC框架出来，将自己所学发挥出来，踏踏实实去做。一来可以更加了解框架底层代码如何运行以及相关原理，二来可以作为面试的一个Project，毕竟如果总是做CURD的网站可是会和博主一样被面试官吐槽项目没有技术含量哦。

  1. 哈佛大学公开课：构建动态网站
    这里之所以乱入一个视频课，是因为博主认为这个视频真的很适合那些想要进入PHP Web后台开发的同学了解相关的很多概念，这个老师真的很不错。
  2. 《PHP和MySQL Web开发》，LUKE WELLING / Laura Thomson
    这本书被称为PHP的圣经，我想是因为它分别对PHP和MySQL做了深入浅出的分析，不仅介绍PHP和MySQL的一般概念，而且对PHP和MySQL的Web应用做了较全面的阐述，并包括几个经典且实用的例子。不过看这书之前要有HTML、CSS、JavaScript相关的基础哦。
  3. 《深入PHP 面向对象、模式与实践》，[美] Matt Zandstra
    这本书分成三大部分，分别是PHP面向对象的内容，设计模式和关于PHP的一些最佳实践。它适合PHP的进阶学习，如果没有PHP开发基础，不建议看这本书。书中的面向对象对于没了解过OOP的同学是一个不过的补习，设计模式的重要性在用到轮子之后就会深刻体会到了。而本书介绍的那些工具也是非常使用的。

Linux

说实话如果从事Web开发的话，个人感觉Linux几乎是一定要好好掌握的。博主面试的时候这个吃了不少亏（Linux的日志清除，定时计划等没答出来），所以建议如果没有特别的Windows需要的话，可以拿Ubuntu Desktop来作为日常用机的操作系统的，这样子可以学习不少东西，毕竟熟能生巧嘛。当然如果是Mac的话就不必了，因为Mac属于Unix。Linux这方面一是要学习系统的使用，二是了解一下Shell脚本，这个面试常考。

  1. 《鸟哥的Linux私房菜》，鸟哥
    学习Linux的入门级读物，讲得很有针对性，实用性非常强，唯一的缺点：太老了！

Java

  1. 《Java语言程序设计》，[美] Y·Daniel Liang
    我都Java就是从这里开始的，感觉这是入门的好书。而且，它适合完全没有编程语言基础的人看，作者字斟句酌，也建议读者不要直接一眼带过。因为概念和原理的阐述，就隐在字里行间，由作者娓娓道来。。
  2. 《Java核心技术·卷1：基础知识》，[美] Cay S·Horstmann / [美] Gary Cornell
    这书同样是入门书，不少同学都在看这本，不过说实话个人更喜欢《Java语言程序设计》。
  3. 《Java编程思想》，[美] Bruce Eckel
    这书基本也可以称上Java中的“圣经”了。里面关注的都是Java最核心的部分。但是建议这本还是有一点Java基础再来看会轻松许多。
  4. 《深入理解Java虚拟机》，周志明
    我觉得既然学习Java，而Java字节码又是运行在Java虚拟机上的，那虚拟机的学习也是必不可少的。而这就是一本很优秀的介绍 Java 虚拟机的书籍。
  5. 《Head First Servlets & JSP》， [美] 巴萨姆 / [美] 塞若 / [美] 贝茨
    本书让一个染指B/S开发不久的菜鸟有机会更深入更详实的理解Servlet与JSP之间的关系，容器在整个HTTP请求—响应过程中所扮演的至关重要的角色，以及JSTL与定制标签“跨时代”的意义。

设计模式

  1. 《图说设计模式》
    本书使用图形和代码结合的方式来解析设计模式；每个模式都有相应的对象结构图，同时为了展示对象间的交互细节，用到时序图来介绍其如何运行。

其他

  1. 《剑指Offer》，何海涛
    毕竟我们还是要从象牙塔中走出去的，这本书剖析了50个典型的程序员面试题，从基础知识、代码质量、解题思路、优化效率和综合能力五个方面系统整理了影响面试的5个要点。建议在大三上学期到下学期之前将它好好看完，题目都做一遍。加上相对的基础和项目，那么Offer收割机非你莫属。
  2. 《经济学原理》，[美] N·格里高利·曼昆
    我觉得不管什么专业的学生，最后都懂一些经济的知识。这本书并没有多么高深，它是我们西方经济学的教材，它生动有趣，整本书洋溢着浓厚的人文热情，没有任何复杂的数学公式，图形和曲线也简化到了极致。相信我，去看它，你不会后悔的。

【转】CGI、FastCGI和PHP-FPM关系图解

Posted on 2017-03-25 | In Technology , PHP |

在搭建 LAMP/LNMP 服务器时，会经常遇到 PHP-FPM、FastCGI和CGI 这几个概念。如果对它们一知半解，很难搭建出高性能的服务器。接下来我们就以图形方式，解释这些概念之间的关系。

基础

在整个网站架构中，Web Server（如Apache）只是内容的分发者。举个栗子，如果客户端请求的是 index.html，那么Web Server会去文件系统中找到这个文件，发送给浏览器，这里分发的是静态数据。

如果请求的是 index.php，根据配置文件，Web Server知道这个不是静态文件，需要去找 PHP 解析器来处理，那么他会把这个请求简单处理，然后交给PHP解析器。

当Web Server收到 index.php 这个请求后，会启动对应的 CGI 程序，这里就是PHP的解析器。接下来PHP解析器会解析php.ini文件，初始化执行环境，然后处理请求，再以规定CGI规定的格式返回处理后的结果，退出进程，Web server再把结果返回给浏览器。这就是一个完整的动态PHP Web访问流程，接下来再引出这些概念，就好理解多了。

队列模型

队列的基本操作是Enqueue（入队），它是在表的末端（叫做队尾（rear））插入一个元素，还有Dequeue（出队），它是删除（或返回）在表的开头（叫做队头（front））的元素。

队列的数组实现

关于队列的循环实现，有两件事情要警惕。第一，检测队列是否为空是很重要的，因为当队列为空时一次Dequeue操作将不知不觉地返回一个不确定的值。第二，某些程序设计人员使用隐式的方法来计算队列的大小，这时候也要注意，因为存在特殊情形。

数据库学习笔记三（关系数据库设计）

Posted on 2016-11-30 | In Technology , Database |
  1. 函数依赖：设R（U）是一个关系模型、X、Y为属性U全集的子集、若对于R（U）中任意一个可能的关系r中、不可能存在两个元组在X上的属性相等、而在Y属性上不等的情况、则称“X函数决定Y”或称“Y函数依赖X函数”。记作：X->Y。
    自己理解：关系模型中若有两个属性X、Y、当X的值一样时、不可能出现两个相同的Y的值、则说明X能决定Y、Y依赖于X。
    • 完全依赖：X->Y、并且X的任何真子集都不能实现X(真)->Y。
    • 部分函数依赖：X->Y、并且X存在真子集实现X(真)->Y。
    • 传递函数依赖：R(u)，X->Y、Y不属于X，Y!->X，Y->Z则成Z对X传递函数依赖
    • 平凡非平凡函数依赖：X->Y(Y属于X)则称平凡函数依赖、否则称非平凡函数依赖。
  2. 在关系模型中，我们将属性不具有任何子结构这个思想形式化。如果该域的元素被认为是不可分的单元，那么这个域就是原子的。如果R的所有属性的域都是原子的，我们称R属于第一范式（First Normal Form，1NF）
  3. 一般地，如果β⊆ α，则形如α->β的函数依赖是平凡的，其中，α，β表示属性集。
  4. 我们用F+符号来表示F集合的闭包，也就是能够从给定F集合推导出的所有函数依赖的集合。例如：
    f={a->b,b->c,a->d,e->f}
    由a可直接得到b和d，间接得到c，则a的闭包就是{a,b,c,d}
  5. 我们能搭到的比较满意的范式之一是Boyce-Codd范式（Boyce-Codd Normal Form，BCNF）。它消除所有基于函数依赖能够发现的冗余。具有函数依赖集F的关系模式R属于BCNF的条件是，对于F+中所有形如α->β的函数依赖（其中α⊆ R且β⊆ R），下面至少有一项成立：
    • α->β是平凡的函数依赖（即，β⊆ α）
    • α是模式R的一个超码
  6. 分解不属于BCNF的模式的一般规则。设R为不属于BCNF的一个模式。则至少存在一个非平凡的函数依赖α->β，其中α不是R的超码。我们在设计中用以下两个模式取代R：
  • （α∪β）
  • （R-（β-α））
    当我们分解不属于BCNF的模式时，产生的模式中可能有一个或多个不属于BCNF。在这种情况下需要进一步分解，其最终结果是一个BCNF模式集合。
  1. 具有函数依赖集F的关系模式R属于第三范式（third normal form）的条件是：对于F+中所有形如α->β的函数依赖（其中α⊆ R且β⊆ R），下面至少有一项成立：
    • α->β是平凡的函数依赖（即，β⊆ α）
    • α是模式R的一个超码
    • B - α中的每个属性A都包含于R的一个候选码中。
  2. 给定关系模式r（R），如果r（R）的每一个满足F的实例也满足f，则R上的函数依赖被r上的函数依赖集逻辑蕴涵（logically implay）
    假设我们给定关系模式r（A,B,C,G,H,I）及函数依赖集：
    A->B
    A->C
    CG->H
    CG->I
    B->H
    那么函数依赖
    A->H
    被逻辑蕴涵。

    • 当实体在某个属性上没有值时使用空值。空值可以表示“不适用”，即该实体的这个属性不存在值。或者用来表示属性值未知。未知的值可以是缺失的，或不知道的。
  3. 映射基数，或基数比率，表示一个实体通过一个联系集能关联到的实体的个数。
    • 一对一。A中的一个实体至多与B中的一个实体相关联，并且B中的一个实体也至多与A中的一个实体相关联
    • 一对多。A中的一个实体可以与B中的任意数目实体相关联，而B中的一个实体至多与A中的一个实体A相关联
    • 多对一。A中的一个实体至多可以与B中的一个实体相关联。而B中的一个实体可以与A中的任意数目实体相关联
    • 多对多。A中的一个实体可以与B中的任意数目的实体相关联。而B中的一个实体也可以与A中的任意数目的实体相关联
  4. 参与约束，如果实体集E的每个实体都参与到联系集R的至少一个联系中，实体集E在联系集R中的参与称为全部的，如果实体集E只有部分实体参与到R的联系中，实体集E到联系集R的参与称为是部分的。
  5. 弱实体集，没有足够的属性以形成主码的实体集称为弱实体集，有主码的实体集称为强实体集。
